## 🔄 Fréquence de Collecte

Tous les médias sont scrapés **toutes les 4 heures** par le GLUE_JOB :

**Horaires UTC** : 00:03, 04:03, 08:03, 12:03, 16:03, 20:03

**Blocs temporels** :
- 00:00 - 04:00 UTC
- 04:00 - 08:00 UTC
- 08:00 - 12:00 UTC
- 12:00 - 16:00 UTC
- 16:00 - 20:00 UTC
- 20:00 - 00:00 UTC

Chaque manchette est associée au bloc 4h durant lequel elle est apparue en Une.

---

## 📝 Données Collectées par Média

Pour chaque manchette, on collecte :

| Champ | Description |
|-------|-------------|
| `media_id` | Identifiant du média (JDM, LAP, CNN, etc.) |
| `country_id` | Région (QC, CAN, USA) |
| `headline_start` | Début du bloc 4h |
| `headline_stop` | Fin du bloc 4h |
| `headline_minutes` | Temps exact d'apparition en Une (en minutes) |
| `title` | Titre de la manchette |
| `body` | Corps de l'article (si disponible) |
| `author` | Auteur (si disponible) |
| `url` | URL de l'article |
| `insert_datetime` | Date d'insertion dans le datawarehouse |
| `tag` | Version du scraper |

## 📈 Pondération et Normalisation

Les médias ont des **cycles de publication différents** :
- Quotidiens papier : 1 édition/jour (Le Devoir)
- Web 24/7 : MAJ continues (CNN, TVA)

Pour comparer équitablement, le système Vitrine **normalise** le temps en Une :

```
Temps pondéré = headline_minutes × facteur_normalisation

Où facteur_normalisation dépend du média et de sa fréquence de MAJ
```

Cela permet de calculer des scores de saillance comparables entre JDM (quotidien) et CNN (continu).

## 🔍 Qualité des Données (SONAR)

Le système **SONAR** surveille quotidiennement la qualité du scraping pour chaque média :

**Métriques suivies** :
- Nombre de frontpages scrapées
- Nombre de unique headlines
- Articles avec body
- Articles avec title
- Articles avec author
- Nombre total de mots

Ces métriques sont visualisées sous forme de **heatmaps** publiées chaque mercredi sur Slack.

Voir `docs/ARCHITECTURE.md` pour plus de détails sur SONAR.
